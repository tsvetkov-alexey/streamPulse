# StreamPulse

Проект, использующий публичный TwitchApi, сделанный по архитектурной методологии FSD (feature-sliced design)

# Авторизация
Есть возможность входа только через twitch-аккаунт

Главная мысль: фронт не должен сам обменивать code на токены Twitch, 
поскольку для этого нужен client_secret, а его нельзя хранить на фронте, 
т.к его можно просто подсмотреть в консоли разработчика и в итоге
скомпрометировать токены. Twitch прямо пишет, что Authorization Code Flow 
предназначен для приложений с сервером, который может безопасно хранить client_secret.
